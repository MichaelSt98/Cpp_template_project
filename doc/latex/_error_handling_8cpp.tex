\hypertarget{_error_handling_8h_autotoc_md158}{}\doxysubsection{Error handling}\label{_error_handling_8h_autotoc_md158}
Errors fall into two categories\+:
\begin{DoxyItemize}
\item {\bfseries{Syntax errors}} occurring when a statement is not valid according to the grammar of the C++ language
\item {\bfseries{Semantic errors}} occurring when a statement is syntactically valid, but does not do what the programmer intended
\end{DoxyItemize}\hypertarget{_error_handling_8h_autotoc_md159}{}\doxysubsubsection{Assert statements}\label{_error_handling_8h_autotoc_md159}
An assert statement is a preprocessor macro that evaluates a conditional expression at runtime. If the conditional expression is true, the assert statement does nothing. If the conditional expression evaluates to false, an error message is displayed and the program is terminated. This error message contains the conditional expression that failed, along with the name of the code file and the line number of the assert.\hypertarget{_error_handling_8h_autotoc_md160}{}\doxyparagraph{assert()}\label{_error_handling_8h_autotoc_md160}
Include $<$cstdlib$>$ for {\ttfamily assert()}, operating at runtime, which comes with a small performance cost that is incurred each time the assert condition is checked, which can be disabled by defining the {\bfseries{N\+D\+E\+B\+UG}} macro.\hypertarget{_error_handling_8h_autotoc_md161}{}\doxyparagraph{static\+\_\+assert()}\label{_error_handling_8h_autotoc_md161}
Another type of assert is {\ttfamily static\+\_\+assert$<$condition, diagnostic\+\_\+message$>$}, designed to operate at compile time.

{\bfseries{Attention}} There is no chance for cleanup after terminating with exit() or assert()!
