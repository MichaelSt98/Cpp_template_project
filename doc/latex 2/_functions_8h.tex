\hypertarget{_functions_8h_autotoc_md102}{}\doxysubsubsection{Pass by value}\label{_functions_8h_autotoc_md102}
By default, non-\/pointer arguments in C++ are passed by value. When an argument is {\bfseries{passed by value}}, the argument\textquotesingle{}s value is copied into the value of the corresponding function parameter. Therefore the original argument can not be modified by the function!\hypertarget{_functions_8h_autotoc_md103}{}\doxyparagraph{Pros}\label{_functions_8h_autotoc_md103}

\begin{DoxyItemize}
\item Arguments can be anything
\item Arguments are never changed by the function (prevents possibly unwanted side effects)
\end{DoxyItemize}\hypertarget{_functions_8h_autotoc_md104}{}\doxyparagraph{Cons}\label{_functions_8h_autotoc_md104}

\begin{DoxyItemize}
\item Copying classes and structs can incur a significant performance penalty
\end{DoxyItemize}\hypertarget{_functions_8h_autotoc_md105}{}\doxyparagraph{When to use}\label{_functions_8h_autotoc_md105}

\begin{DoxyItemize}
\item When passing fundamental data type and enumerators, and the function does not need to change the argument.
\end{DoxyItemize}\hypertarget{_functions_8h_autotoc_md106}{}\doxyparagraph{When not to use}\label{_functions_8h_autotoc_md106}

\begin{DoxyItemize}
\item When passing structs or classes (including std\+::array, std\+::vector, and std\+::string) 
\end{DoxyItemize}
